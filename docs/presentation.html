The Problem

GraphRAG over-retrieves

  • Vector + graph retrieval boosts recall — but returns duplicate, overlapping, marginally-relevant context.
  • The LLM gets far more than it needs → more tokens, more cost, no guaranteed accuracy gain.
  • Challenge: transform high-recall retrieval into a smaller, precise, higher-quality evidence set before generation.
  • Not just dedup — similar passages may carry different facts, dates, exceptions. Keep complementary evidence, drop redundancy.

The Core — Context Optimizer

High recall in, precise & small out — deterministically

  • 2-hop graph fusion: expand on companies named in the query and bridge entities found in the evidence (e.g. acquirer → its auditor).
  • Semantic dedup: drop redundant passages, keep ones adding new figures/dates.
  • MMR: diversity-aware selection (coverage, not 5 copies of one fact).
  • Sentence compression to an adaptive budget: tight for single-hop, larger for many-company aggregation.
  • Deterministic → 0 extra inference tokens. The token savings are real, not shifted into another LLM call.
Experimental Controls

Fair, reproducible, traceable

  • Same LLM (Gemini 2.5 Flash, temp 0), output cap, embedding model, and one shared core prompt.
  • Judge grades evidence-blind vs the reference — identical for every pipeline (no context-length bias).
  • Support & citations scored deterministically; full per-question token accounting (system/question/context/output/total).
  • Dev/test split · held-out eval run 3× (GraphRAG grade stdev ≈ 0) · every result stamped with its git commit.

Challenges & Lessons

What we learned

  • Deterministic optimization > LLM-based filtering — reproducible and zero extra inference tokens.
  • Grade evidence-blind — showing the judge variable-length context biased grading; grading vs the reference alone is fair + reproducible.
  • Adaptive budgets matter on heterogeneous data — single-fact lookups vs many-company aggregation.
  • Two-hop fusion unlocks bridge questions — the second entity is discovered from evidence, not given.
Generalization & Roadmap

Beyond this dataset

  • Works across heterogeneous content — structured financial facts + long-form governance — with one adaptive strategy.
  • Next: question-type routing, community summaries for broad topics, subgraph caching, learned reranking.
  • Everything reproducible from the public repo; TigerGraph Savanna serves both graph and vector retrieval.
